<WAR 빌드>
프로젝트 폴더로 이동 후 프로젝트 빌드 명령어 입력:
    ./gradlew build
빌드 폴더 이동:
    cd build
    cd libs
WAR 빌드 파일 확인:
    server-0.0.1-SNAPSHOT.war
WAR 빌드 압축 해제:
    jar -xvf server-0.0.1-SNAPSHOT.war

build 패키지 내부의 libs 패키지에 WEB-INF 패키지 확인
WAR(Web Application Archive)파일은 WAS(Web Application Server)에 배포될 떄 사용되는 파일이다
JAR 파일은 JVM 위에서 실행되고 WAR 는 WAS 위에서 실행된다
WAR 는 정적 리소스, 클래스 파일을 모두 포함하며 JAR 에 비해 구조가 복잡하다 그리고 WAR 구조를 지켜야 한다

<WAR 구조>
WEB-INF
    classes: 실행 클래스 모음
    lib: 라이브러리 모음
    web.xml: 웹 서버 배치 설정 파일(생략 가능)
index.html: 정적 리소스

즉, WEB-INF 패키지 및 하위 패키지는 자바 클래스, 라이브러리, 설정 정보가 들어가고 WEB-INF 를 제외한 나머지 영역은 HTML, CSS 같은 정적 리소스가 사용되는 영역이다

* Tomcat 서버가 종료된 상태여야 한다
Tomcat 패키지 - webapps 패키지 내부에 WAR 빌드 파일(server-0.0.1-SNAPSHOT.war)을 넣어주고 ROOT.war 로 이름을 바꾼다
이후 Tomcat 을 다시 실행하고, 웹 사이트에 서블릿 경로를 입력한다(http://localhost:8080/test)
Tomcat - logs- catalina.out 을 실행해서 로그 확인 가능
